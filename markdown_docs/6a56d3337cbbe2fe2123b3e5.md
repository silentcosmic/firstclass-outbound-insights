# 深度分析：新加坡企业AI规模化为何步履维艰？

新加坡企业正积极推进人工智能（AI）部署。然而，一份最新行业洞察报告指出，在扩大AI应用规模的过程中，这些企业正面临一系列挑战，尤其在技术基础和治理层面存在显著障碍。
这项覆盖全球14个国家、逾4600名IT领袖的调查，深入剖析了企业在规模化AI应用时遭遇的困境。研究发现，新加坡企业在AI雄心方面表现突出，已有75%的企业正在部署或试行“智能体AI”（agentic AI）解决方案。
然而，报告也揭示了深层次的结构性问题：78%的新加坡IT领袖认为，实时数据基础设施的不足是阻碍AI落地的主要障碍；73%的受访者指出数据所有权碎片化问题；另有73%的企业面临AI管理技能与专业知识的匮乏。这清晰表明，新加坡AI规模化的瓶颈并非源于缺乏进取心，而是根植于基础设施、数据治理和专业人才方面的基础性缺陷。
基础设施的挑战也同样拖慢了智能体AI的部署步伐。新加坡高达95%的领导者表示，他们在数据基础设施和数据质量方面遇到或预见到困难；同样有95%的受访者指出在传统系统集成方面存在挑战；另有93%的人提及大语言模型（LLM）的可靠性问题。因此，超过73%的智能体AI项目陷入停滞，其中半数甚至已彻底放弃。
治理层面是另一道关键障碍。近期一项行业调研，针对全球292位高管和492位知识工作者进行了访问，结果显示全球仅约半数组织已建立明确的AI使用政策。
不明确的治理规范正导致高风险的安全行为。该研究发现，员工经常与AI工具共享敏感的公司及个人数据，这其中包括：工作邮件或内部信息（51.4%）、人力资源相关信息（35.8%）、机密公司文件（29.1%）、健康相关信息（25%）、个人身份信息（20.3%），以及登录凭据和密码（16.3%）。
风险不仅限于数据共享，员工还将AI工具接入关键的内部系统，例如文件或云存储（38.4%）、电子邮件收件箱（37.6%）、企业协作工具（37.4%）以及日程表（30.7%）。
这些挑战因企业对员工AI使用情况可见度有限而进一步加剧。一份针对1200名IT和网络安全专业人员的最新研究发现，企业对员工AI使用情况的了解十分有限，仅有48%的新加坡企业表示能完全掌握员工在工作中使用的AI工具和大型语言模型（LLM）。
此前提及的行业研究报告也呼应了这些担忧，IT领袖们强调了透明度的迫切需求。超过五分之四（86%）的新加坡IT领袖将“持续且实时的业务可见性”列为首要业务优先事项。
这种需求也使得数据主权和数据溯源性成为关注焦点。86%的受访者认为有效管理数据主权至关重要，同时82%的受访者高度重视有效的数据溯源和追踪能力。
在这些基础设施挑战之下，投资优先级也正悄然转变。86%的新加坡领导者将数据流（data streaming）列为投资重点，紧随其后的是AI和机器学习解决方案（85%），以及数据管理与治理（90%）。
数据流指的是数据在生成时即进行持续、实时的移动和处理，而非存储后进行批处理。这类平台被视为赋能AI系统的关键基础设施，因为它们能提供对可信、情境化数据的即时访问，从而有效克服传统系统集成难题并提升大语言模型的可靠性。
尽管面临诸多挑战，新加坡政府正加紧推动AI的普及，尤其是在中小型企业（SMEs）中。
在五月份举办的“亚洲科技 x 新加坡”大会上，新加坡数字发展与信息高级政务部长陈凯豪宣布了一系列举措，旨在帮助企业自信地采纳AI，同时增强网络韧性，并为新加坡的数字基础设施进行未来规划。这些举措包括在《数字企业蓝图》（DEB）框架下，与Grab及RSM Stone Forest IT建立新的合作关系。
这些合作关系将为中小型企业提供实用的指导、工具以及技术专长，助力它们信心十足地部署AI，并同步强化网络安全韧性。
Grab将与新加坡资讯通信媒体发展局（IMDA）合作，推出“Grab中小企业AI计划”，通过一系列举措惠及10,000家餐饮（F&B）、电商和零售领域的中小型企业。该计划将提供免费在线培训视频、免费大师班和网络研讨会，并包含一个与新加坡科技设计大学共同开发的为期两天的结构化AI课程。
与此同时，RSM将推出“RSM网络安全2中小企业计划”，为2,000家中小企业提供免费网络钓鱼模拟演练，以帮助它们识别组织漏洞并强化网络防御态势。
于2024年启动的《数字企业蓝图》（DEB）明确了新加坡在支持中小企业数字化转型和采纳AI赋能解决方案方面的战略。迄今为止，已有超过26,000家中小企业从中受益。
过去几年中，新加坡企业的AI采纳率显著加速。根据新加坡资讯通信媒体发展局（IMDA）的一项研究，中小企业的AI采纳水平从2023年的4.2%跃升至2024年的14.5%，实现了三倍增长。非中小企业的AI采纳比例也大幅提升，从2023年的44%上升约20个百分点，达到2024年的62.5%。
