# AI原生云基础设施新格局：Railway融资1亿美元挑战传统巨头

旧金山云平台Railway，在未投入任何市场营销费用的情况下，悄然积累了200万开发者。近日，该公司宣布完成1亿美元B轮融资。随着人工智能应用需求的激增，传统云基础设施的局限性日益凸显，Railway的此次融资正逢其时。
本轮融资由TQ Ventures领投，FPV Ventures、Redpoint和Unusual Ventures跟投。这项投资彰显了Railway作为AI时代涌现的关键基础设施初创公司之一的地位，它敏锐地抓住了开发者对亚马逊网络服务（AWS）和谷歌云等传统平台复杂性及高成本的不满。“随着AI模型在代码编写方面愈发精进，越来越多的人开始追问那个由来已久的问题：我的应用程序究竟在哪里、如何运行？”Railway 28岁的创始人兼首席执行官杰克·库珀（Jake Cooper）在一次专访中表示。“上一代云原语已经显得缓慢且过时，现在随着AI加速一切，团队根本无法跟上节奏。”
对于这家在云计算行业走出非传统路径的公司而言，此次融资无疑是一次巨大的加速。此前，Railway在A轮融资中获得了Redpoint的2000万美元投资，总融资额仅为2400万美元。如今，该公司每月处理超过1000万次部署，通过其边缘网络处理超过1万亿次请求——这些指标已经与规模更大、资金更充足的竞争对手不相上下。
在AI编码助手时代，三分钟部署已不可接受
Railway的价值主张基于一个简单观察：开发者用于部署和管理软件的工具，是为更慢的时代设计的。使用行业标准的Terraform进行一次标准的构建与部署周期，通常需要两到三分钟。这种曾经可以容忍的延迟，现在已成为一个关键瓶颈，因为像Claude、ChatGPT和Cursor这样的AI编码助手可以在几秒钟内生成可运行的代码。库珀表示：“当神一般的智能随手可得，能在三秒内解决任何问题时，那些整合的系统就变成了瓶颈。以前对人类来说，在10秒或更短时间内完成部署很酷，但现在对AI代理来说，这只是基本要求。”
该公司声称其平台能在不到一秒的时间内完成部署，速度足以跟上AI生成代码的节奏。客户报告称，与传统云提供商相比，开发者效率提高了十倍，成本节约高达65%。
这些数据直接来源于企业客户，而非内部基准测试。G2X（一个服务于10万联邦承包商的平台）首席技术官丹尼尔·洛巴顿（Daniel Lobaton）测量后发现，迁移到Railway后，部署速度提升了七倍，成本降低了87%。他的基础设施费用从每月1.5万美元降至约1000美元。洛巴顿说：“以前在我们的旧基础设施上需要一周的工作，现在在Railway一天就能完成。如果我想启动一项新服务并测试不同的架构，在旧设置上会花费很长时间。而在Railway，我可以在两分钟内启动六项服务。”
放弃谷歌云、从零开始自建数据中心的‘大胆’决策
Railway与Render、Fly.io等竞争对手的区别在于其垂直整合的深度。2024年，该公司做出了一个不同寻常的决定：完全放弃谷歌云，并自建数据中心。此举呼应了艾伦·凯（Alan Kay）那句名言：“真正严肃对待软件的人，应该自己制造硬件。”库珀表示：“我们希望以一种能带来差异化体验的方式来设计硬件。对网络、计算和存储层的完全控制，使我们能够实现极快的构建和部署循环，这种速度让我们能够以‘代理速度’运行，同时保持业内最流畅的体验。”
事实证明，这种方法带来了回报。在近期影响主要云服务商的广泛中断期间，Railway始终保持在线。
这种端到端的控制使得Railway的定价可以比超大规模服务商便宜约50%，比新兴云初创公司便宜三到四倍。Railway按实际计算使用量以秒计费：每千兆字节内存秒收费0.00000386美元，每vCPU秒收费0.00000772美元，每千兆字节存储秒收费0.00000006美元。空闲虚拟机不收取任何费用——这与传统云模式形成鲜明对比，传统模式下客户需要为预置容量付费，无论是否使用。库珀指出：“传统观点认为，大公司凭借规模经济能够提供更优惠的价格。但当他们对云中通常闲置的虚拟机收费，而我们则为机器内部构建了更高的密度，这就带来了巨大的机会。”
30名员工如何创造数千万美元年收入的平台
Railway凭借一支仅30名员工的团队，实现了数千万美元的年收入，这样的员工人均收入比，即使对于成熟的软件公司而言也堪称卓越。公司去年收入增长了3.5倍，并持续保持每月15%的环比增长。库珀强调，此次融资是战略性的，而非必要。“我们默认可以生存；我们没有理由筹集资金，”他说。“我们融资是因为看到了巨大的加速机会，而不是因为我们需要生存。”
该公司直到去年才聘请了第一位销售人员，目前仅有两名解决方案工程师。Railway的200万用户几乎都通过口碑发现这个平台——开发者将这个真正好用的工具口耳相传。库珀回忆道：“我们基本上做了标准的工程之事：如果你建造了它，他们就会来。某种程度上，他们确实来了。”
从个人项目到财富500强部署：Railway异乎寻常的企业扩张
尽管拥有草根开发者社区，Railway已在大中型组织中取得显著进展。该公司声称，财富500强企业中有31%正在使用其平台，尽管部署范围从公司级基础设施到个人团队项目不等。
著名客户包括忠诚度计划公司Bilt；Intuit旗下的GoCo子公司；TripAdvisor的Cruise Critic；以及MGM Resorts。由Y Combinator支持的Kernel是一家为1000多家公司提供AI基础设施的初创公司，其整个面向客户的系统都在Railway上运行，每月费用为444美元。Kernel首席技术官拉斐尔·加西亚（Rafael Garcia）表示：“在我之前的公司Clever（以5亿美元出售），我曾有六名全职工程师专门管理AWS。现在我总共有六名工程师，他们都专注于产品。Railway正是我在2012年梦寐以求的工具。”
对于企业客户，Railway提供SOC 2 Type 2合规和HIPAA准备等安全认证，并可应要求提供业务伙伴协议（BAA）。该平台提供单点登录身份验证、全面的审计日志，并支持通过“自带云”配置在客户现有云环境中部署。
企业定价从定制级别开始，包含多种附加项，例如延长日志保留（每月200美元）、HIPAA BAAs（1000美元）、带服务水平目标（SLO）的企业支持（2000美元）以及专用虚拟机（10000美元）。
这家初创公司挑战亚马逊、谷歌及新一代云竞争对手的大胆战略
Railway进入了一个竞争激烈的市场，其中不仅有超大规模云提供商——亚马逊网络服务（AWS）、微软Azure和谷歌云平台（GCP），还有Vercel、Render、Fly.io和Heroku等专注于开发者的平台。
库珀认为，Railway的竞争对手分为两大阵营，但两者都未能完全致力于AI所需的新型基础设施模式。他观察到：“超大规模服务商拥有两个相互竞争的系统，他们还没有完全投入新模式，因为他们传统的收入来源仍在持续盈利。他们拥有一大笔现金流，来自于那些预置了虚拟机但可能只使用了10%却仍需支付全部费用的客户。如果他们并没有真正需要，那么他们究竟有多大的兴趣去全力以赴地开发一种全新的体验呢？”
针对初创公司竞争对手，Railway通过覆盖完整的L3基础设施堆栈来差异化。“我们不只是容器；我们拥有虚拟机原语、有状态存储、虚拟专用网络、自动化负载均衡，”库珀说。“我们将所有这些都包装在一个极其易用的用户界面中，并带有代理原语，让代理能够以1000倍的速度运行。”
该平台支持PostgreSQL、MySQL、MongoDB和Redis等数据库；提供高达256TB的持久存储，每秒输入/输出操作超过10万次；并支持部署到横跨美国、欧洲和东南亚的四个全球区域。企业客户每个服务可扩展至112个vCPU和2TB内存。
投资者押注AI将创造比现有软件多千倍的未来
Railway的融资反映了投资者对受益于AI编码革命公司的广泛热情。随着GitHub Copilot、Cursor和Claude等工具成为开发者工作流中的标准配置，代码的编写量——以及运行这些代码所需的基础设施——正在急剧扩张。库珀预测：“未来五年内上线软件的数量将是前所未有的，比现有软件多出千倍。所有这些都需要运行在某个地方。”
该公司已直接与AI系统集成，构建了库珀所称的“能让Claude接入、调用部署并自动分析基础设施的循环”。Railway于2025年8月发布了模型上下文协议（Model Context Protocol）服务器，允许AI编码代理直接从代码编辑器部署应用程序和管理基础设施。库珀说：“开发者的概念正在我们眼前消融。你不再需要是工程师才能进行工程工作——你只需要批判性思维和系统分析能力。”
Railway将如何利用1亿美元和“零”市场经验走向全球
Railway计划利用这笔新资金扩大其全球数据中心版图，将团队规模从30名员工扩大，并首次在公司成立五年以来建立库珀所描述的“完善的市场推广运营”。库珀解释道：“我的一位导师曾说，当你能够改变业务轨迹时就应该融资。我们已经构建了无限扩展所需的所有底层结构；阻碍我们的仅仅是‘表达’。2026年将是我们登上世界舞台的一年。”
该公司的投资者名单可谓开发者基础设施领域的“名人录”。天使投资者包括GitHub联合创始人汤姆·普雷斯顿-沃纳（Tom Preston-Werner）；Vercel首席执行官吉列尔莫·劳奇（Guillermo Rauch）；Cockroach Labs首席执行官斯宾塞·金博尔（Spencer Kimball）；Datadog首席执行官奥利维耶·波梅尔（Olivier Pomel）；以及Linear联合创始人乔里·拉洛（Jori Lallo）。
Railway此次扩张的时机，恰逢硅谷许多人认为软件制造方式正在发生根本性转变。编码助手不再是实验性的新奇事物，它们已成为数百万开发者日常依赖的必备工具。每一行AI生成的代码都需要运行的场所，而库珀认为，现有巨头过于固守其现有商业模式，未能充分抓住这一机遇。
Railway能否将开发者的热情转化为持续的企业级采纳，仍是一个悬而未决的问题。云基础设施市场充斥着许多充满希望的初创公司，它们未能打破亚马逊、微软和谷歌的垄断。但库珀，这位曾在Wolfram Alpha、Bloomberg和Uber担任软件工程师，并于2020年创立Railway的人，似乎并未因其宏大抱负而动摇。他说：“五年内，Railway将成为软件创建和进化的首选之地，仅此而已。即时部署，无限扩展，零摩擦。这是值得争夺的最高奖励，没有比这更大的了。”
对于一家通过反其道而行之（无营销、无销售团队、无风险炒作）而建立了估值1亿美元业务的公司而言，真正的考验才刚刚开始。Railway用了五年时间证明开发者会自行找到更好的“捕鼠器”。接下来的五年将决定世界其他地区是否准备好加入。
