# 赛富时（Salesforce）推出全新智能助理Slackbot，重塑企业协作AI格局

赛富时（Salesforce）于日前推出全面重构的职场助理Slackbot。这款产品已从单纯的通知工具，蜕变为高管们所描述的强大AI代理，能够深度搜索企业数据、高效起草文件，并代表员工执行各项任务。
新版Slackbot现已面向Business+和Enterprise+客户全面开放，标志着赛富时迄今为止最积极的举措，旨在将Slack定位为新兴“智能代理AI”运动的核心——即软件代理与人类协同工作，共同完成复杂任务。此次发布也正值赛富时努力向投资者证明，人工智能将助推而非淘汰其现有产品。
赛富时联合创始人兼Slack首席技术官Parker Harris指出：“Slackbot不仅仅是另一个协同AI或智能助手，它是通向智能代理企业的前沿门户，由赛富时赋能。”
从三轮车到保时捷：赛富时彻底重构Slackbot
Harris直言不讳地阐释了新旧Slackbot的显著区别：“旧版Slackbot就像一辆小三轮车，而新版Slackbot则好比一辆保时捷。”
自Slack早期便已存在的旧版Slackbot，主要执行基础算法任务，例如提醒用户添加同事到文档、建议频道归档以及发送简单通知。而新版本则运行在一个全新的架构之上，该架构围绕大语言模型和复杂的搜索能力构建，能够访问赛富时记录、谷歌云端硬盘文件、日历数据以及多年来的Slack对话。
Harris进一步解释：“两者截然不同。旧版Slackbot基于算法且相对简单。新版Slackbot是全新的，它以大语言模型（LLM）和一个极其强大的搜索引擎为核心，并连接到第三方搜索引擎及第三方企业数据。”
尽管进行了根本性的技术革新，赛富时选择保留了“Slackbot”这一品牌。Harris表示：“用户都知道Slackbot是什么，所以我们希望延续这个名称。”
Anthropic的Claude驱动新版Slackbot，未来或引入更多AI模型
新版Slackbot运行在Anthropic的大语言模型Claude之上，这一选择部分是出于合规性要求。Slack的商业服务在FedRAMP中等认证下运营，以服务美国联邦政府客户。Harris表示，当Slack开始构建新系统时，Anthropic是“唯一能提供合规大语言模型”的供应商。
然而，这种独占性不会持续。Harris透露：“今年，我们将支持更多的供应商。我们与谷歌关系良好。Gemini令人惊叹——性能卓越，成本效益高。所以我们将在某些方面使用Gemini。”他补充说，OpenAI也仍是潜在选择。
Harris呼应了赛富时首席执行官Marc Benioff的观点，即大语言模型正在商品化：“你们听Marc谈论过大语言模型是商品、是民主化的。我称它们为CPU。”
在敏感的训练数据问题上，Harris的立场 unequivocal：赛富时不会使用客户数据训练任何模型。他解释说：“模型没有任何安全保障。如果我们用你我之间的某些机密对话来训练模型，我不想让Carolyn知道——如果我将其训练进大语言模型，就没有办法说只有你能看到答案，而Carolyn不能。”
赛富时内部实验：8万员工测试Slackbot，效果显著
赛富时已在内部测试新版Slackbot数月，并将其推广至所有8万名员工。据Slack首席营销官Ryan Gavin透露，结果令人瞩目：“这是赛富时历史上采用速度最快的产品。”
内部数据显示，三分之二的赛富时员工已尝试新版Slackbot，其中80%的用户持续定期使用。内部满意度高达96%——这是Slack发布过的所有AI功能中最高的一次。员工报告每周节省2到20小时。
此次采纳在很大程度上是自然发生的。Gavin说：“大约五天后，我们的员工就创建了一个名为‘最值得学习的Slackbot提示’的Canvas。人们开始自发地添加内容。现在这个Canvas中的提示已超过250个。”
赛富时首席用户体验研究员Kate Crotty发现，73%的内部采用是由社交分享而非自上而下的强制驱动的。她表示：“每个人都在那里互相帮助学习和交流技巧。”
Slackbot如何将分散的企业数据转化为高管洞察
在产品演示中，Slack产品体验设计师Amy Bauer展示了Slackbot如何综合来自多个来源的信息。在一个例子中，她要求Slackbot分析来自试点项目的客户反馈，上传一张使用仪表板的图片，并让Slackbot关联定性和定量数据。
Bauer解释说：“这就是Slackbot真正发挥价值的地方。它不仅仅是简单地读取图片，它实际上是在查看图片，并将其与刚才为我生成的洞察进行比较。”
随后，Slackbot可以查询赛富时系统，找出有开放交易的企业账户，这些账户可能是早期访问的理想候选者，从而创建Bauer所称的“一个绝佳的理由和推进计划”。最后，它能将所有这些信息综合到Canvas（Slack的协作文档格式）中，并查找相关利益方在日历上的空闲时间以安排评审会议。
Bauer表示：“在此之前，我们一直以一对一的方式与Slackbot互动。但现在我能够利用这个洞察，将其生成到Canvas中，这是一个共享工作区，我可以在其中与Slackbot一起迭代、完善，或者与我的团队分享。”
Slack首席产品官Rob Seaman表示，Canvas的创建展示了产品的发展方向：“这正在内部调用Slack Canvas工具，以实际撰写一份共享文档。但这预示了Slackbot的未来走向——我们最终将添加额外的第三方工具调用。”
MrBeast公司试用Slackbot：员工每日节省90分钟
MrBeast的母公司Beast Industries是赛富时的试点客户之一。该公司首席信息官Luis Madrigal在发布会上分享了他的经验。
Madrigal表示：“作为一个部署企业技术超过二十年的人，这几乎是最简单的部署之一。基础设施已经到位。Slack作为一种实施方案，其企业工具——启用Slackbot和Slack AI功能，就像让我的团队进行审查，快速通过安全审查一样简单。”
Madrigal称其安全团队“相当迅速地”批准了——这在企业AI部署中并不常见——因为Slackbot仅访问每个独立用户已有权限查看的信息。他说：“鉴于你们为Slackbot设置的所有防护措施，使其独特定制，仅限于每个用户可访问的信息、他们所属的对话、Slack房间和Slack频道——这使得我的安全团队能够相当迅速地批准。”
Beast Industries的一名员工，Beast Games营销主管Sinan表示，他“至少”每天节省90分钟。另一名员工，创意总监Spencer，将其描述为“一个在我不在时依然保持关注的助手”。
其他试点客户包括Slalom、reMarkable、Xero、Mercari和Engine。Engine运营高级副总裁Mollie Bodensteiner称Slackbot为“我们团队的绝对‘混乱驯服者’”，估计每天能帮她节省约30分钟，“仅仅是消除了上下文切换”。
Slackbot与微软Copilot、谷歌Gemini：企业AI霸权之战
此次发布使赛富时与集成在Teams和更广泛Microsoft 365套件中的微软Copilot，以及跨Workspace集成的谷歌Gemini展开直接竞争。当被问及Slackbot与这些替代方案的区别时，Seaman强调了上下文和便利性。
Seaman表示：“对我们的客户和用户而言，它最强大的地方在于其近距离——它就近在你的Slack中。其中自然内置了巨大的便利性优势。”
高管们认为，更深层次的优势在于Slackbot已经理解用户的工作，无需额外设置或培训。公司声明指出：“大多数AI工具无论谁使用都听起来大同小异。它们缺乏上下文，无法捕捉细微之处，并强迫你频繁切换工具来完成工作。”
Harris更直接地表示：“如果你曾有过那种神奇的AI体验——我认为ChatGPT就是一个很好的例子，从消费者角度来看体验极佳——那么Slackbot正是我们在企业中所做的事情，旨在成为一个深受喜爱的员工超级代理，就像人们喜爱使用Slack一样。”
Amy Bauer强调了这种无摩擦的体验。她表示：“Slackbot天生就根植于你在Slack中的上下文和数据。因此，当你继续在Slack中工作时，Slackbot会变得更好，因为它以你正在进行的工作为基础。终端用户无需任何设置，也无需任何配置。”
赛富时雄心勃勃的计划：让Slackbot成为控制所有其他代理的“超级代理”
赛富时将Slackbot定位为Harris所称的“超级代理”——一个最终能够协调组织内其他AI代理的中心枢纽。
Harris表示：“每个公司都将拥有一个员工超级代理。Slackbot实质上正在汲取Slack所做事情的魔力。我们认为Slackbot，并且对此感到非常兴奋，将成为那个角色。”
这一愿景延伸到已在Slack中推出的第三方代理。上个月，Anthropic发布了Claude Code for Slack的预览版，允许开发者直接在聊天线程中与Claude的编码能力互动。OpenAI、谷歌、Vercel等公司也已为该平台构建了代理。
Seaman在新闻发布会上指出：“部署到Slack上的大多数新应用都是代理。这证明了人类与代理在Slack中共同存在并协作解决问题的潜力。”
Harris描述了一个未来，Slackbot将成为MCP（Model Context Protocol）客户端，能够利用整个软件生态系统中的工具——类似于开发者工具Cursor的工作方式。他说：“Slack可以成为一个MCP客户端，而Slackbot将是这个枢纽，利用世界上所有这些工具，其中一些将是令人惊叹的代理。”
但Harris也提醒不要对多代理协调过度承诺。他说：“我仍然认为我们处于单代理世界。2026财年将是我们开始看到更多协调的一年。但我们将以客户成功为中心来做这件事，而不是展示和谈论‘我有1000个代理协同工作’，因为我认为那是不现实的。”
Slackbot免费，但赛富时数据访问费用或影响部分客户
Slackbot对Business+和Enterprise+计划的客户不收取额外费用。Gavin确认：“客户无需支付额外费用。如果他们属于这些计划之一，他们就能获得Slackbot。”
然而，一些企业客户可能面临与赛富时更广泛数据战略相关的其他成本压力。由于API访问费用的提高，第三方应用程序与赛富时数据交互的成本可能会上涨，其影响会波及整个软件供应链，导致CIO们看到价格上涨。
Fivetran首席执行官George Fraser曾警告，赛富时API访问定价政策的变化可能对依赖赛富时作为记录系统的企业产生实际后果。Fraser在最近的一份报告中表示：“他们可能无法使用Fivetran将其数据复制到Snowflake，而不得不使用赛富时Data Cloud。或者他们可能会发现无法通过ChatGPT与其数据交互，而不得不使用Agentforce。”
赛富时已将此次定价变化解释为行业标准做法。
Slackbot当前功能、未来几周更新与路线图
新版Slackbot已于近日开始推出，并将于2月底前覆盖所有符合条件的客户。Bauer在采访中确认，移动端将于3月3日前完成可用性部署。
部分功能仍在开发中。日历读取和可用性检查已随发布上线，但实际预订会议的功能“将在几周后推出”。图像生成目前尚不支持，尽管Bauer表示“这是我们未来正在考虑的功能”。
当被问及与HubSpot和微软Dynamics等竞争性CRM系统的集成时，赛富时代表在采访中拒绝提供具体细节，但他们承认这个问题触及了关键的竞争差异点。
赛富时押注未来工作形态：一个聊天窗口
Slackbot的推出是赛富时的一项重要押注，即未来企业工作将是对话式的——员工将越来越多地倾向于通过自然语言而非传统的软件界面与AI互动。
Harris用“无需思考”和“成为优秀主人”等原则描述了Slack的产品理念。他表示，目标是让Slackbot主动呈现信息，而不是要求用户去寻找。
Harris指出：“对我来说，一个重要的启示是，大语言模型应用于非结构化信息时，其能力令人难以置信。如果你是Slack用户，如果你的公司使用Slack——Slack中的价值是难以置信的。因为你在讨论工作、共享文档、做出决策，但作为人类，你无法像大语言模型那样深入挖掘并真正获得同样的价值。”
展望未来，Harris预计界面本身将超越纯粹的对话式。他说：“我们正在某种程度上达到纯对话式界面的饱和点。我认为我们将开始看到代理构建最符合你意图的界面，而不是试图在对话界面中呈现与你意图匹配的内容。”
微软、谷歌以及越来越多的AI初创公司也正在进行类似的押注——赢得企业AI竞争的关键在于将其嵌入员工已经使用的工具中，而非引入另一个需要学习的应用程序。这场争夺职场智能隐形层的竞赛现已全面展开。
对赛富时而言，此次产品发布的影响远超单一产品。在华尔街经历动荡一年，并持续面临AI是否威胁其核心业务的质疑后，公司正在押注Slackbot能证明相反——每天数千万在Slack中聊天的用户并非脆弱点，而是一种不可动摇的优势。
赛富时匹兹堡客户经理Haley Gault在一个下雪的早晨偶然发现了新版Slackbot，她用一句话概括了这种转变：“我真的无法想象在没有这类工具的公司工作。这已经成为我的工作方式。”
这正是赛富时所期望的。
