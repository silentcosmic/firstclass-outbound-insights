# 企业AI智能体编排：挑战在于部署而非平台，多数仍将聊天机器人误作智能体

在对101家企业的调研中，智能体编排正加速整合至模型提供商平台——其中Anthropic的Claude以显著优势领先。企业选择此类平台主要基于底层模型的吸引力，并以可靠的多步骤执行能力作为评判标准。然而，当前雄心远超现实：多数已部署的“智能体”仍是聊天机器人的简单封装；企业期望的控制平面旨在通过混合模式规避供应商锁定；同时，对令牌消耗进行实时财务控制仍是罕见实践。
本期研究深入探讨了企业智能体编排现状：企业主要运行于哪些平台，选择背后的驱动因素是什么，其优化目标何在，对智能体控制结构的预期如何，以及——最关键的揭示——已部署的“智能体”实际编排程度以及成本控制的严密性。
核心发现在于编排愿景与实际落地之间存在显著差距。企业正快速整合到主要模型平台：Anthropic的Claude是40%企业的首选平台，是第二名平台（微软，18%）的两倍多，随后是OpenAI（13%）。平台选择受“模型引力”驱动——即与最先进的基础模型原生对齐（21%），而衡量成功的标准则是可靠的多步骤执行（任务完成可靠性32%，多步骤工作流管理28%）。然而，当被要求诚实评估其智能体组合时，71%的企业表示，其已部署的“智能体”中，只有四分之一或更少是真正的多步骤编排工作流，而非单指令聊天机器人封装；仅有10%的企业越过了半数门槛。这表明，编排层正在迅速构建，但其本应运行的编排组合却远未成形。
这一差距也塑造了企业正在构建的架构。到2026年底，明确多数企业（51%）预期采用混合控制平面——即提供商原生与外部编排相结合——只有6%的企业期望将控制权完全交给提供商托管服务。这是因为，供应商锁定（35%）是他们最担心控制权若存在于模型提供商内部将带来的风险。投资也随之倾斜：智能体工作流工具投入居首（34%），其次是安全与权限强制（25%）。然而，财务控制始终滞后——超过四分之一（27%）的企业缺乏实时方法在账单生成前阻止失控的智能体。
研究方法
出海 · 头等舱开展了此次调研，作为其持续研究系列的一部分，本次聚焦于企业智能体编排。受访者筛选为拥有100名或以上员工的企业（N=101），数据来源于2026年6月进行的单次调查；由于这是一次性而非多月汇总样本，报告呈横截面分析，不推断月度趋势。
按组织规模划分，样本在各企业层级中分布均匀：100-499名员工、2,500-9,999名员工和50,000名以上员工各占21%；10,000-49,999名员工和500-2,499名员工各占19%。按职责划分，受访者均为资深且具备采购决策影响力的专业人士：产品和项目经理（15%）、CIO/CTO/CISO（13%）、顾问和咨询师（13%），以及数据、AI和工程总监/副总裁等，另有18%归为“其他”职能。在采购方面，81%的受访者是AI解决方案的推荐者、影响者或最终决策者（66%为推荐者/影响者，15%为最终决策者）。技术/软件是最大的行业类别，占44%，其次是金融服务（17%）和医疗保健/生命科学（8%）。
共101名受访者，样本规模足以提供具有合理信度的方向性解读，尽管其为自选样本，并非概率样本。
发现1：编排运行于模型提供商平台
Anthropic的Claude居首；开放框架作用有限
我们询问了企业目前主要使用哪种智能体编排平台。结果集中于主要模型提供商——尤其是其中一家。
关于解读这些份额的说明：如方法论部分所述，受访者是自选的，并且此问题要求他们选择一个主要平台——因此，这些数据衡量的是在AI活跃技术决策者这一自选群体中，哪个平台引领了每家企业的部署。以这种方式构建的样本可能与基于支出的市场份额数据存在显著差异，且每项研究都会抽取具有自身公司规模组合的独立样本，因此不同调查中的供应商数据不应进行比较。请将这些份额解读为该群体目前主要编排押注的写照，而非市场占有率。
模型平台占据主导地位。Anthropic、微软、OpenAI、谷歌和亚马逊共计占据约80%的部署份额（101家中的81家），而那些在工程讨论中占据核心地位的开放框架（LangChain/LangGraph）和定制内部构建则仅占个位数。Anthropic的领先地位（40%，是第二名平台的两倍多）反映了发现2中的“模型引力”选择逻辑：企业正在选择与其希望构建的领先模型最接近的编排环境。如同上一次智能体安全研究中的安全供应商，那些在技术圈内定义类别的工具，尚未成为企业部署的集中点。一小部分（3%）企业甚至尚未进行任何编排。
受访者对他们所使用的平台整体评价为3.94分（满分5分，109人回答），其中“性价比”得分3.94，而“实施便捷性”得分最低，为3.85——这使得编排工具的满意度在我们五项跟踪指标中处于较低位置，仅高于评估工具。接近4分的评分，来自96%计划在一年内改变其编排方法的受访者，这表明一种暂时性的接受：这些平台足以在当下运行，但尚未好到足以停止对更优解的探索。这些评分与普遍的变更意图并存；这是一个企业更多是容忍而非钟爱的层面。
发现2：模型引力驱动平台选择
基础模型而非工具本身决定平台选择
我们询问了哪些因素对编排平台的选择影响最大。最大单一因素是底层模型的吸引力——尽管灵活性和开发便捷性紧随其后。
“模型引力”占据主导，是Anthropic平台领先的侧面解释：企业选择与其标准化部署的领先模型最接近的编排环境。但紧随其后的因素使情况变得复杂——跨模型和工具的灵活性（17%）以及开发便捷性（17%）表明企业也希望避免被这一选择所束缚，这预示了发现6中的锁定担忧。安全与权限（14%）和总拥有成本（11%）构成了一个务实的采购逻辑。性能（延迟/内存）以4%的比例位居末位，这提醒我们，在当前的采纳阶段，主要制约因素是模型匹配度和选择性，而非原始速度。
发现3：核心任务是可靠的多步骤执行
企业以是否完成任务来衡量编排成效
我们询问了企业最看重什么——他们衡量编排成功的首要指标。可靠性和多步骤工作流管理占据主导；面向开发者和用户的指标则居于次要地位。
任务完成可靠性（32%）和多步骤工作流管理（28%）合计占59%的响应（101家中的60家）：在企业看来，当编排能够可靠地将任务通过多个步骤执行完成时，它才算成功。开发者生产力（17%）固然重要但居于次要地位——这与它在框架讨论中的突出程度相反——而终端用户体验（9%）则是一个次要考虑，这与编排更多是内部执行问题而非用户体验问题相符。这种“可靠性优先”的标准，恰恰使得“多数智能体仍停留在聊天机器人层面”的发现尤为尖锐：企业将成功定义为可靠的多步骤执行，但其大多数已部署的“智能体”却根本未进行多步骤工作。
这种局限性并非普遍存在。按组织规模划分样本，77%的较小企业表示其四分之一或更少的智能体能进行真正的多步骤工作，而较大企业的这一比例为62%。较大企业在真正的多步骤部署方面显然更进一步；这种对聊天机器人的依赖现象，方向性上而言，是中型市场企业普遍存在的状况。
发现4：整合、生产化与内部构建
未来一年三大战略举措不分伯仲
我们询问了企业预计未来12个月内其编排策略将发生哪些重大变化。三大举措位居前列，几乎平分秋色。
排名前三的举措——构建内部控制（25%）、标准化单一框架（24%）以及将智能体从沙盒推向生产环境（23%）——在统计上没有显著差异，共同揭示了一个趋势：企业正从实验阶段转向运营整合。他们希望减少框架数量，增加生产环境的部署，并对控制层拥有更多所有权；只有4%的企业预期没有变化。值得注意的是，尽管在发现1中平台集中度较高，但企业对定制化内部控制平面的渴望依然显著——企业在标准化使用模型提供商平台的同时，也计划通过自有控制逻辑对其进行封装，这正是发现6中明确提出的混合姿态。
发现5：投资流向工作流工具
工具和权限引领支出；监控滞后
我们询问了明年哪些与编排相关的投资增长最多。智能体工作流工具居首，其次是安全和权限强制。
工作流工具居首（34%）是发现3中“可靠性和多步骤优先”这一重点在预算方面的体现：资金正流向能够可靠地串联各个步骤的机制。安全与权限强制（25%）和扩展基础设施（20%）紧随其后——这些是将智能体从沙盒推向生产环境（发现4中的战略举措）所需的投资。监控和调试投入较少（11%），另有11%表示预算持平。对工具、权限和扩展的重视程度超过纯粹的可观测性，表明企业正在投入资金构建和强化编排，而不仅仅是监控其运行状况。
发现6：控制平面将是混合模式——原因在于锁定风险
企业预计将控制权分配给提供商和自有层面
我们询问了企业预计到2026年底，智能体的主要控制平面将设在哪里，以及如果控制权归属于模型提供商平台，他们最担心什么。明确多数企业预期采用混合模式——而供应商锁定是主要原因。
混合控制是压倒性的主流预期（51%），只有6%的企业期望将控制权完全交给提供商托管服务。综合来看，混合、定制和外部抽象选项——所有至少部分将控制权置于提供商之外的架构——总计占88%（101家中的89家）。当被问及提供商驻留控制的风险时，原因直接浮现：供应商锁定以35%（101家中的35家）的比例居首，领先于安全和权限限制（28%）以及跨模型和工具的灵活性不足（21%）。这种模式呼应了上一期“不信任模型能够自我监管”的立场——在此，企业将在提供商平台上构建，但拒绝完全受其治理。混合控制平面是对他们最担忧的锁定风险的架构对冲。
6月份数据显示企业偏好混合控制平面，这标志着与早期相比的转变。在4-5月的调查中（N=145），只有34%的企业预期采用混合控制平面，而更多企业（12%）预期将控制权完全交给提供商托管服务。这两个快照尚未衡量出确凿的长期趋势——但对话的方向是明确的：倾向于保留控制权。
锁定风险也作为首要担忧因素首次出现。在4-5月的调查中，主要担忧是安全和权限限制（32%），锁定风险位居第二（24%）；到6月，两者互换了位置。对提供商平台的担忧似乎正从其是否安全成熟到其是否可被替换。
发现7：多数“智能体”仍未真正智能化
企业承认多数部署仍是聊天机器人封装
我们要求企业诚实评估其智能体组合：在已部署的“智能体”中，有多少是真正的多步骤编排工作流，而非简单的单指令聊天机器人封装。这一答案是本期研究的决定性发现。
这正是本报告核心所揭示的差距。结合底部两个区间，71%的企业（101家中的72家）表示，其已部署的“智能体”中，只有四分之一或更少是真正经过编排的——仅有10%（101家中的10家）跨过了半数门槛。早期发现中记录的雄心——模型提供商平台、可靠性优先的成功指标、生产环境部署、深思熟虑的控制架构——远超已部署的现实，后者仍然绝大多数是披着智能体外衣的单指令助手。这与其说是一个矛盾，不如说是一份路线图：平台、预算和战略的落实，恰恰是因为编排组合仍然非常薄弱。未来研究的悬念在于，现实能否迅速缩小与雄心之间的差距。
发现8：财务控制仍属被动
仅少数企业能在账单生成前阻止失控智能体
最后，我们询问了企业如何对智能体令牌消耗实施财务控制——即自主循环可能耗尽预算而无人干预的风险。多数企业依赖于原生上限或事后监控；实时编程控制仍是例外。
超过四分之一的企业（27%）承认，他们没有实时、编程的方式来阻止一个智能体在导致超预算账单前失控——他们只能事后从日志中获知。另有32%的企业完全依赖于其主要平台内置的原生上限和节流机制，这种控制的好坏仅取决于提供商的工具，并与发现6中的锁定担忧相关联。那些构建定制网关（23%）或利用跨模型路由进行成本套利（19%）的企业，正在将令牌消耗视为一个需要确定性控制的工程问题。如同编排成熟度一样，财务控制也是一个运营现实滞后于雄心的领域：智能体正更快地推向生产环境，而围绕它们的成本控制平面却尚未完全建成。
值得注意的是，公司规模也呈现出差异：在员工人数少于2,500人的企业中，约三分之一（34%）仅对智能体支出进行被动控制，而较大企业的这一比例为20%——这些数据具有方向性，但与多数智能体仍停留在聊天机器人层面的状况相符。中型市场企业在成本控制薄弱的预算下运行着成熟度最低的智能体。
总结：编排层已初具规模，而真正的智能体尚待成熟
拥有100名或更多员工的组织所描述的编排策略正迅速整合，但成熟度提升缓慢。他们正在标准化模型提供商平台——Anthropic的Claude以40%的比例领先——选择是基于底层模型的吸引力，并以可靠的多步骤执行来衡量成功。投资正流向工作流工具和权限，战略目标是整合框架并将智能体推向生产环境，而他们预期的控制平面是刻意设计的混合模式，因为供应商锁定是他们最担心的问题。
然而，诚实的自我评估却戳破了这一雄心。71%的企业表示，其已部署的“智能体”中，只有四分之一或更少是真正经过编排的，仅有10%的企业越过了半数门槛，并且超过四分之一的企业无法实时阻止失控的智能体。编排层——包括平台、预算和控制架构——正在提前构建，而其本应运行的真正编排组合却远未成形。鉴于本次调查有101名受访者，且仅为6月单次调查，这更多被视为一个清晰的方向性信号而非精确衡量：企业在大多数智能体尚未真正发挥编排作用之前，就已经决定了如何编排智能体。未来调查的疑问在于，实际部署能否缩小与雄心之间的差距——或者说，这种对聊天机器人的依赖现象是否比路线图预想的更为顽固。
本报告基于对101名合格企业受访者（100+名员工）的调查回复，数据来源于2026年6月进行的单次调查。由于这是一次性而非多月汇总样本，结果呈方向性解读，不作为已确认趋势。受访者包括产品和项目经理、CIO、CTO和CISO、顾问和咨询师，以及数据、AI和工程总监/副总裁等，横跨技术/软件、金融服务、医疗保健及其他行业。
